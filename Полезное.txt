В качестве "боевого сервера" создал на www.amazon.com/ec2 инстанс Ubuntu 16. Добрый (и богатый) Amazon даёт "начинающим" программистам бесплатно ресурсы в рамках программы Free Tier на 1 год. Через год (или раньше) можно этот аккаунт убить и зарегистрировать новый, на новую почту (например, gmail.com). Этих ресурсов достаточно, чтобы ваш сервер был запущен целый год круглосуточно. Грабли, которые я собрал:
1) Подключение к свежему серверу Ununtu немного отличается от видео автора. Никаких ключей, как на видео, генерировать не надо. Надо использовать тот ключ, который вам выдал Amazon после запуска инстанса (например, max.pem):
ssh -i ~/.ssh/max.pem ubuntu@16.17.18.190
Ключ, естественно, нужно положить в каталог .ssh в домашнем каталоге.
После того, как что-то спросит при первом подключении, нужно набрать yes (y или д - недостаточно!)
2) Сервера Amazon по умолчанию имеют английскую локаль, поэтому лучше сразу переключить на русскую. Из командной строки это сделать можно. Это нужно для того, чтобы на следующем этапе встал postgresql.
3) Устанавливаем postgres:
sudo apt-get install postgresql-client postgresql postgresql-contrib
4) Очень удивился, что на дефолтном образе Ubuntu нет Java. Устанавливаем (погуглите как, например, с помощью webupd8.org)

Чтобы остановить веб-сервер, введите:
sudo systemctl stop nginx

Чтобы запустить остановленный веб-сервер, введите:
sudo systemctl start nginx

Чтобы остановить и снова запустить службу, введите:
sudo systemctl restart nginx

Если вы просто вносите изменения в конфигурацию, во многих случаях Nginx может перезагружаться без отключения соединений. Для этого введите:
sudo systemctl reload nginx

По умолчанию Nginx настроен на автоматический запуск при загрузке сервера. Если вы не хотите этого, вы можете отключить такое поведение с помощью следующей команды:
sudo systemctl disable nginx

Чтобы перезагрузить службу для запуска во время загрузки, введите:
sudo systemctl enable nginx

Устанавливаем NGINX

sudo amazon-linux-extras install nginx1.12
редактируем конфигурацию:

sudo vi  /etc/nginx/nginx.conf
меняем имя пользователя. Было "nginx" стало "ec2-user".

обновляем раздел location так:

location / {
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_pass http://localhost:8080;
}

location /img/ {
    alias /home/ec2-user/uploads/;
}
открываем доступ пользователю ec2-user к папке сервера nginx:

sudo chown -Rf ec2-user:ec2-user /var/lib/nginx
запускаем сервер:

sudo service nginx start
если нужен перезапуск сервера:

sudo nginx -s reload
если нужно посмотреть логи nginx:

sudo vim /var/log/nginx/error.log
